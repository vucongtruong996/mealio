# 3. Layered Architecture

Every service follows:

```text
Controller
    ↓
Service
    ↓
Repository
    ↓
Database
```

Responsibilities:

## Controller

* Receive HTTP requests
* Validate input
* Convert request DTO
* Return response DTO

Must NOT:

* Access Repository
* Contain business logic

---

## Service

Responsible for:

* Business rules
* Validation
* Domain operations
* Transaction boundaries

Must NOT:

* Return JPA Entities
* Know HTTP details

---

## Repository

Responsible for:

* Database access only

Must NOT:

* Contain business rules

---

# 5. Entity Rules

Entities represent database tables.

Requirements:

* @Entity
* @Table
* UUID primary key
* createdAt
* updatedAt

Never expose entities directly.

Example:

```text
MenuItemEntity
```

---

# 6. DTO Rules

Separate request and response DTOs.

Example:

```text
CreateMenuItemRequest

UpdateMenuItemRequest

MenuItemResponse
```

Never reuse Entity as API object.

---

# 7. Mapping Rules

DTO conversion belongs to Mapper classes.

Example:

```text
MenuItemMapper
```

Responsibilities:

* Entity → Response
* Request → Entity

Controllers should not perform mapping.

---

# 8. Service Rules

Service interface:

```text
MenuItemService
```

Implementation:

```text
MenuItemServiceImpl
```

Controllers depend on interfaces.

---

# 9. Repository Rules

Repositories extend:

```text
JpaRepository<Entity, UUID>
```

Custom queries only when necessary.

Prefer derived query methods.

Example:

```text
findByCategory()

findByDeletedAtIsNull()
```

---

# 10. Exception Handling

Global exception handler required.

Example:

```text
GlobalExceptionHandler
```

Custom exceptions:

```text
ResourceNotFoundException

ValidationException

BusinessException
```

Never expose stack traces.

---

# 11. Validation

Use Jakarta Validation.

Example:

```text
@NotBlank

@NotNull

@Positive

@Size
```

Validation occurs at Controller layer.

---

# 12. Transactions

@Transactional belongs to Service layer.

Controllers must never manage transactions.

---

# 13. Logging

Use SLF4J.

Do:

* Application start
* Business events
* Errors

Do NOT:

* Log passwords
* Log JWT tokens
* Log sensitive health data

---

# 14. UUID Standard

All entities use UUID.

Generated by application.

Never use auto increment IDs.

---

# 15. Audit Fields

Every table contains:

```text
id

created_at

updated_at

deleted_at
```

Soft delete preferred.

---

# 16. Configuration

Environment specific configuration:

```text
application.yml

application-local.yml

application-dev.yml

application-prod.yml
```

Secrets never stored in Git.

---

# 17. Testing Strategy

## Unit Tests

Test:

* Service
* Mapper
* Utility classes

Mock:

* Repository
* External services

---

## Integration Tests

Use:

Testcontainers

Test:

* Repository
* Database
* REST endpoints

# 19. Clean Code Principles

* Small methods
* Single Responsibility
* Constructor Injection
* No Field Injection
* No Static Business Methods
* No Magic Numbers
* No Hardcoded Values

---

# 20. AI Agent Coding Rules

AI-generated code must:

* Follow defined architecture
* Respect layer boundaries
* Keep business logic in Domain layer
* Never bypass service contracts
* Never access databases directly from transport layers
* Generate tests
* Generate documentation
* Avoid framework-specific coupling in business logic
* Use dependency injection
* Follow naming conventions

AI agents must optimize for maintainability over brevity.
---

# 21. Git Commit Convention

Pattern:

type(scope): description

Examples:

feat(menu): add menu item creation

fix(order): handle inventory timeout

docs(api): add REST API standard

test(menu): add service unit tests
